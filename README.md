**Tutoriel GitHub**

Ce tutoriel est destiné aux néophytes de GitHub. Il vous permettra de comprendre comment créer votre compte GitHub, comment l'optimiser, et comment initialiser ou dupliquer un dépôt (repository).

**1\. Création et Configuration de votre compte GitHub** 

Ouvrez votre navigateur web et recherchez "github.com" sur Google.   
 

<img width="521" alt="GitHub - Recherche sur Google" src="https://github.com/user-attachments/assets/c7dc178b-4ee2-488f-a146-f8617abe0eaf" />


  
Cliquez sur le lien officiel GitHub pour accéder à la plateforme. 

Sur la page d'accueil de GitHub, vous verrez une section pour commencer votre inscription.   
  
Entrez votre adresse email et cliquez sur "Sign up for GitHub" ou "Sign up" en haut à droite. Remplissez les champs requis (Email, Mot de passe, Nom d'utilisateur, Pays/Région).

**3\. L'interface du repository** 

**Le repository ou dépot :** C'est l'espace en ligne où vous organisez et stockez l'intégralité des fichiers de votre projet sur GitHub. Il ne sert pas uniquement de simple dossier de stockage ; il intègre un système de suivi qui enregistre chaque modification apportée à votre travail, permettant de consulter l'historique complet des versions et de faciliter la collaboration entre plusieurs contributeurs sur le même projet.

**Le README :** Le fichier est la **vitrine de votre projet** sur GitHub : c'est le premier élément que les visiteurs voient en arrivant sur votre dépôt. Rédigé en Markdown, il sert à présenter votre projet, expliquer son objectif, fournir des instructions d'utilisation, et même inclure des images, offrant ainsi une compréhension immédiate et une bonne première impression de votre travail.

**4\. Le fork sur Github**

Un **"Fork"** (qui signifie littéralement "fourche" ou "duplication") est l'action de créer une **copie personnelle et indépendante** d'un repository qui ne vous appartient pas, sous votre propre compte GitHub. Cette duplication vous permet de modifier, d'expérimenter ou de développer le projet original sans affecter l'œuvre de départ, tout en conservant un lien pour potentiellement proposer vos améliorations à l'auteur original si vous le souhaitez.

![GitHub - faire un fork](https://github.com/user-attachments/assets/b2dab5f1-074c-40fb-b575-e20caf2a2732)



  
**GitHub Pages**  
Au-delà du simple stockage de code, GitHub offre une fonctionnalité très pratique appelée **GitHub Pages**. Elle vous permet de transformer le contenu de votre dépôt en un **site web public et accessible**. C'est idéal pour héberger un site personnel, un portfolio, ou la documentation d'un projet directement depuis votre dépôt.
